# region-grab プラグイン 設計

- 日付: 2026-06-29
- 種別: 同梱プラグイン（`plugins/region-grab.js`）＋ ブリッジ1メソッド追加
- 関連: [webview-extension-runtime](2026-06-28-webview-extension-runtime-design.md) / [session-list-readable](2026-06-28-session-list-readable-design.md) / [screens-and-plugins](2026-06-28-screens-and-plugins-design.md)

## 目的

モバイルの WebView 上で、Claude Code 拡張のチャット欄・セッション一覧・各種パネル等の
**「DOM 上はテキストだが、モバイルの選択操作が暴れて掴めない UI テキスト」** を、
**フローティングボタン → 矩形ドラッグ** で囲って即コピーできるようにする。

OCR は使わない。対象が DOM テキストである以上、矩形に重なるテキストノードを直接収集する方が
**誤認識ゼロ・軽量・ネイティブ改変ほぼ不要**。OCR（ピクセル認識）はターミナル描画/canvas/画像など
「DOM にテキストが無い対象」に必要な技術で、本機能の対象外。

### なぜプラグイン層か（VS Code 拡張では作れない）

| 必要なこと | プラグイン（注入層） | VS Code 拡張 |
| --- | --- | --- |
| 全画面オーバーレイ / 常駐 FAB を置く | ○ ページコンテキストに注入 | ✗ 公式 UI API（コマンド/View/Webview パネル等）の範囲しか出せない |
| 他拡張（Claude Code）の webview 内テキストを読む | ○ 全フレームに注入され各フレーム DOM を読める | ✗ 他 webview はサンドボックス iframe で読めない |
| クロスフレーム連携 | ○ 各フレームのインスタンスが postMessage で協調 | ✗ |

拡張が読めるのは原則 `TextDocument`（実エディタ本文）のみで、本機能の対象である**パネル/chrome の UI テキスト**には届かない。
よって本機能はプラグイン層が唯一の実装層。アプリの思想「WebView の外側/注入層からモバイル摩擦を回収する」とも一致する。

## 構成

単一の `.js`（`@all-frames true` / `@run-at document-start`、既存3本と同じ作法）。
全フレームに同一コードが注入され、**役割をフレーム位置で分岐**する。

- **トップフレームのみ**: FAB 描画・選択オーバーレイ・矩形ドラッグ・結果のコピーとトースト。
- **全フレーム共通**: 自フレームのスクリーン座標原点の把握、矩形ブロードキャスト受信、テキスト収集と返信。

ネイティブ追加は **`CcBridge.setClipboard(text: String)` の1メソッドのみ**
（JS 側は `navigator.clipboard.writeText` フォールバックを持つ。HTTPS=セキュアコンテキストのため
トップフレームでは JS だけで動く見込みだが、サンドボックス webview からの書き込み拒否に備えてブリッジを正とする）。

## クロスフレーム座標（実装の山場）

別オリジンの子 webview 内テキストも拾うための肝。

1. **原点伝播（上→下）**: 親は子 iframe 要素の `getBoundingClientRect()` を読める（中身がクロスオリジンでも
   *要素自体*は親 DOM）。各フレームは自分のスクリーン座標原点を知っており、
   `子原点 = 親原点 + iframeRect.topLeft`（スクロール考慮）を `postMessage` で各子へ伝える。再帰でツリー全体に伝播。
   トップ原点は (0,0)。レイアウト変化に追従するため定期 or リサイズ/スクロール時に再伝播。
2. **矩形ブロードキャスト（収集時）**: 選択完了でトップが**矩形（スクリーン座標）**を全フレームへ配信。
   各フレームは自分の原点でローカル viewport 座標へ変換しヒットテスト。
3. **断片返信（下→上）**: 各フレームは採用テキストを**スクリーン座標の (top,left) 付き**でトップへ返信。

> 補足: 親が触れるのは iframe *要素*の矩形のみ（中身の DOM はクロスオリジンで不可読）。
> 各フレーム内の DOM は、そのフレームに注入された自インスタンスだけが読む。役割分担はこの境界に沿う。

## テキスト収集（各フレーム内）

- `TreeWalker(SHOW_TEXT)` でテキストノードを走査。空白のみ・非表示（`offsetParent`/可視性）ノードは除外。
- 各ノードの `Range.getClientRects()` がローカル矩形と交差するものを採用（**MVP は交差ノード丸ごと**。部分文字クリップはしない）。
- 採用断片に**スクリーン座標 (top,left)** を付与して返信。

### 連結（トップ側）

- 全フレームの断片を集約し **(top → left) でソート**。
- 縦の隙間が行高相当を超える断片間は **改行**、近接は **空白** で連結。重複除去（同一テキスト×近接座標）。
- 結果を `setClipboard` でコピー。

## UI / 操作フロー

- **FAB**: 半透明の小ボタン。**トップフレームのみ・定位置固定（ドラッグ無し）**。
  `⋮`（switcher 起動ボタン: `position:fixed; left:0; bottom:22%; width:30px; height:84px`）の**直上**に置く:
  `position:fixed; left:0; bottom:calc(22% + 84px + 8px); width:30px;`。z-index は `⋮`/オーバーレイと整合させる。
- **タップ** → 選択モード ON、全画面の透明オーバーレイを最前面に展開。
- **ドラッグ** → 破線の矩形を描画。**指を離す** → 収集 → コピー → **自前トースト「コピーしました」**（ネイティブトースト不要、focus-hud 風の自己描画オーバーレイ）→ モード解除。
- モード中の他タップはオーバーレイが吸収して VS Code 側へ流さない（誤操作防止）。空矩形/0件はトーストで「テキストなし」を出して解除。

## エラーハンドリング

- `setClipboard` 失敗時は `navigator.clipboard.writeText` を試し、それも失敗ならトーストでエラー表示（クラッシュさせない）。
- フレームからの返信が一定時間（例: 300ms）来ない場合は、揃った分だけで確定（サンドボックス/応答不能フレームを待ち続けない）。
- `document.write`/再描画で FAB が消えた場合は MutationObserver + 周期ポーリングで再注入（session-list-readable と同様の冪等再注入）。

## テスト

- **純ロジックを純関数に切り出す**（フレーム非依存）:
  - 矩形×テキスト矩形の交差判定
  - 断片の (top,left) ソート＋改行/空白連結＋重複除去
  - 原点伝播の座標変換（親原点 + iframe オフセット）
  これらを JS の軽量テスト、または既存 `*JsonTest` に倣い JVM 側へ寄せて検証。
- **クロスフレーム伝播・FAB・実機コピー**は実機チェックリスト項目（README 状態欄の運用に合わせ、screens-and-plugins のチェックリストに追記）。

## スコープ外（YAGNI）

- OCR / ピクセル認識（対象が DOM テキストのため不要）。将来ターミナル/画像対応が要れば別プラグインとしてネイティブ OCR 経路を検討。
- 部分文字クリップ（矩形に半分だけ入ったノードの文字単位切り出し）。MVP は交差ノード丸ごと。
- プレビュー/編集してからコピー（「即コピー＋トースト」を採用）。
- FAB のドラッグ移動（定位置固定）。
